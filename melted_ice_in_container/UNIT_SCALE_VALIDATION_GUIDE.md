# RHEO 单位与尺度一致性验证指南

适用范围：
- LAMMPS RHEO（含热-力耦合、相变、流固接触）
- 使用 `units lj` 或其他单位体系时，想确认“结果是否对应目标物理尺寸”

---

## 1. 这个问题到底是什么

很多人会记得一句话：
> 只要单位体系自洽，任何单位体系都可以做同一个仿真。

这句话是对的，但只回答了“**方程是否一致**”，没有自动回答“**是否对应你的目标实物尺寸**”。

你真正关心的是两件事：
1. **单位变换不应改变物理问题本身**（单位不变性）
2. **几何尺寸变化会不会引入真实的尺度效应**（尺寸效应）

两者不是一回事。

---

## 2. 为什么会这样（原理）

## 2.1 单位自洽只保证“同一个无量纲问题”

在 LAMMPS 中，单位体系本质上是内部转换因子。只要你把所有输入量和输出量都做了正确换算，同一个物理问题可以在不同单位体系下复现。

但前提很严格：
- 几何、时间、力学、热学、接触、成键阈值等全部量都要一起换算
- 任何漏换一个阈值，都可能让问题变成另一个问题

## 2.2 是否“尺度一致”由无量纲组决定

对你的场景（流-固-热-相变），至少要看这些无量纲数：

- 力学压缩性：Mach 数
  - $Ma = U / c_s$
- 流动惯性/黏性：Reynolds 数
  - $Re = \rho U L / \mu = UL/\nu$
- 对流/导热：Peclet 数
  - $Pe = UL/\alpha$
- 热扩散时间标度：Fourier 数
  - $Fo = \alpha t / L^2$
- 相变显热/潜热：Stefan 数
  - $Ste = C_p \Delta T / L_h$

如果这些无量纲数不匹配，哪怕单位“自洽”，你也不是在解同一个目标问题。

---

## 3. 怎么做（可落地流程）

下面给出一个可以直接执行的 7 步流程。

## 步骤 1：先定义“验证目标”

先明确你要验证的是哪一类一致性：
- A 类：单位不变性（换单位后结果应重合）
- B 类：尺寸效应（放大/缩小几何后结果是否按理论变化）

建议至少各做一个。

## 步骤 2：选基准尺度并建立映射

在 `units lj` 下，建议先选 3 个基准尺度：
- 长度尺度 $S_L$
- 质量尺度 $S_M$
- 时间尺度 $S_T$

任意量都用：
- $Q_{phys} = Q^* S_Q$

常用导出尺度：
- 速度尺度：$S_U = S_L/S_T$
- 压力尺度：$S_P = S_M/(S_L S_T^2)$
- 动力黏度尺度：$S_\mu = S_M/(S_L S_T)$
- 运动黏度尺度：$S_\nu = S_L^2/S_T$
- 热扩散率尺度：$S_\alpha = S_L^2/S_T$

如果你用“目标声速”来定时标，可用：
- $S_T = S_L c_s^*/c_{s,phys}$

## 步骤 3：把关键无量纲数组装出来

在输入脚本里先算出你当前工况的无量纲数，便于之后对照。示例片段（可粘贴）：

```lammps
# Characteristic scales (example)
variable Lchar equal 9.0      # 例如特征长度（按你的几何定义）
variable Uchar equal 0.01     # 例如特征速度（可用稳态平均或加载速度）
variable nu equal ${eta}/${rho0}
variable alpha equal ${kappa}/(${rho0}*${Cv})

variable Re equal ${Uchar}*${Lchar}/${nu}
variable Ma equal ${Uchar}/${cs}
variable Pe equal ${Uchar}*${Lchar}/${alpha}
variable Pr equal ${nu}/${alpha}

print "[DIMLESS] Re=${Re} Ma=${Ma} Pe=${Pe} Pr=${Pr}"
```

如果有相变，再加：

```lammps
# 需要你定义温差 dT
# variable Ste equal ${Cv}*${dT}/${L}
# print "[DIMLESS] Ste=${Ste}"
```

## 步骤 4：做“单位不变性”测试

目标：确认你在不同单位表达下，解的是同一个问题。

做法：
1. 用当前输入做基准算例（Case-U1）
2. 构造一套等价输入（Case-U2）
   - 改单位表达方式或重标定基准尺度
   - 但保持全部无量纲数一致
3. 输出归一化量后比较：
   - 时间：$t^* = tU/L$ 或 $t^* = t\nu/L^2$
   - 应力：$\sigma^* = \sigma/(\rho U^2)$ 或 $\sigma/(\rho c_s^2)$
   - 温度：$\theta^* = (T-T_0)/\Delta T$
4. 判据：归一化曲线应重合（允许小量数值误差）

## 步骤 5：做“尺寸效应”测试

目标：确认当几何尺寸变为 $\lambda L$ 时，行为是否符合理论。

建议至少做三组：
- $\lambda = 0.5, 1.0, 2.0$

并区分两种实验：

1. 相似性保持实验
- 通过调整边界速度/体力/黏度/导热等，让 Re、Ma、Pe、Fo、Ste 保持不变
- 预期：归一化曲线基本重合

2. 物性保持实验
- 保持材料参数与加载方式不变，仅改几何
- 预期：会出现真实尺寸效应（这是正常且常见的）

## 步骤 6：优先用两类标准算例做“地基验证”

在本仓库可直接用：

1. Poiseuille（通道流）
- 文件：`examples/rheo/poiseuille/in.rheo.poiseuille`
- 可验证：
  - 速度剖面形状相似性：$u/u_{max}$ 对 $y/H$
  - 尺寸放大后，稳态速度与建立时间的尺度关系

2. Taylor-Green（涡衰减）
- 文件：`examples/rheo/taylor-green/in.rheo.taylor.green`
- 可验证：
  - 动能衰减时间尺度与 $L^2/\nu$ 的关系
  - 用 $t^*=t\nu/L^2$ 归一化后曲线重合

先把这两类跑通，再上复杂氧化+接触问题，诊断会更清晰。

## 步骤 7：把流程落到你的氧化输入上

针对 `examples/rheo/oxidation/in.new`，做尺寸变化时，下面这些量必须检查是否同步缩放：

- 几何尺寸：`region box`、`region xialiao`
- 核半径与相互作用截断：`variable cut`、`pair_style rheo ${cut}`、`fix rheo ${cut}`
- 近邻成键/反应长度：
  - `create_bonds ... 1.5`
  - `fix rheo/thermal ... react 1.5 1`
- 固体接触长度参数：
  - `pair_coeff ... rheo/solid ...`
  - `pair_coeff ... smd/tri_surface ...`
- STL 几何本体尺寸（外部 CAD/STL 必须同步）
- 时间相关设置：`timestep`、`run` 步数、输出采样间隔

如果只改了几何，但这些阈值没按同一规则变，最容易出现“看起来单位错了”的假象。

---

## 4. 常见误区（高频）

1. 只改 `units`，不改接触/反应截断
- 结果：接触与成键行为突变

2. 只看原始曲线，不看归一化曲线
- 结果：把正常尺寸效应误判成模型错误

3. 用性能 benchmark 的 scaled 案例当物理尺度验证
- 结果：结论不可靠（性能缩放不等于物理相似）

4. 热学参数只改一半
- 结果：`Pe/Fo/Ste` 偏离，熔化或凝固行为与目标不一致

---

## 5. 建议的验收标准

建议把下面 3 条写成你的项目“通过条件”：

1. 单位不变性通过
- Case-U1 / U2 的归一化结果重合（主指标误差在预设阈值内）

2. 尺寸效应符合理论趋势
- 至少一个基准算例（Poiseuille 或 Taylor-Green）满足解析/半解析趋势

3. 目标工况无量纲一致
- `Re, Ma, Pe, Fo, Ste` 落在目标窗口内

---

## 6. 一页总结（给团队同步）

- 是什么：单位自洽不等于目标尺度一致
- 为什么：是否一致由无量纲组与阈值缩放共同决定
- 怎么做：
  1) 建立尺度映射
  2) 计算无量纲数
  3) 做单位不变性测试
  4) 做尺寸效应测试
  5) 用 Poiseuille/Taylor-Green 先打地基
  6) 再应用到氧化耦合算例

当你把“单位变换”和“几何放缩”分开验证，单位尺度问题会变得非常清楚且可控。
